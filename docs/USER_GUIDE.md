## 1. 安装扩展

### 步骤

1. 下载或克隆本项目到本地：
   ```bash
   git clone https://github.com/yourusername/block-twitter.git
   ```

2. 打开 Chrome，访问 `chrome://extensions/`

3. 右上角开启 **"开发者模式"（Developer mode）**

4. 点击 **"加载已解压的扩展程序"（Load unpacked）**

5. 选择 `block-twitter` 项目的**根目录**（包含 `manifest.json` 的文件夹）

6. 完成！工具栏中出现 block-twitter 图标即为安装成功

> **更新扩展**：修改代码后，在 `chrome://extensions/` 页面点击扩展卡片上的刷新图标，再刷新 X.com 页面即可生效。

## 2. 界面概览

扩展由三个界面组成：

| 界面 | 入口 | 功能 |
|------|------|------|
| **Popup** | 点击工具栏图标 | 查看当前页面已隐藏帖子数量、快速打开设置 |
| **设置页（Options）** | Popup → "Open Settings" | 管理关键词、查看已隐藏用户列表、导入导出数据 |
| **内容覆盖层** | 自动显示在 X.com 页面 | 橙色标记 + Hide User + 🚫 屏蔽(X) 按钮、底部批量操作工具栏 |

## 3. 关键词管理

### 3.1 添加关键词

1. 点击工具栏中的 **block-twitter 图标**
2. 点击 **"Open Settings"**
3. 在输入框中输入关键词，点击 **"Add"**
4. 关键词立即生效，所有已打开的 X.com 标签页无需刷新

### 3.2 关键词匹配规则

| 类型 | 匹配方式 | 示例 |
|------|----------|------|
| **英文关键词** | 完整词匹配（大小写不敏感） | 关键词 `scam` 匹配 `scam`，不匹配 `scammer` |
| **中文 / CJK 关键词** | 子串匹配（无词边界） | 关键词 `垃圾` 匹配任何包含"垃圾"的文字 |
| **含 Emoji 的关键词** | 包含 Emoji 的完整子串匹配 | 关键词 `一直都在 💧` 匹配帖子中的相同内容 |

> **提示**：X.com 将 Emoji 渲染为图片，本扩展会自动提取图片的 `alt` 属性还原 Emoji 进行匹配，无需特殊处理。

### 3.3 搜索关键词

设置页的 "Search keywords..." 输入框支持实时过滤，方便在大量关键词中快速定位。

### 3.4 删除关键词

在关键词列表中，点击对应关键词右侧的 **"Delete"** 按钮即可删除。

### 3.5 关键词上限

最多支持 **1000 个关键词**，超出时会提示错误并拒绝添加。

## 4. 隐藏用户（CSS 隐藏）

### 什么是"隐藏"？

"隐藏"是本地操作，通过 CSS `display: none` 将帖子从视图中移除。**X 平台不感知此操作**，被隐藏的用户不会收到任何通知，你与对方的关注关系也不受影响。

刷新页面或关闭浏览器后，被隐藏的用户数据保存在本地，再次打开页面时帖子依然被隐藏。

### 操作步骤

当帖子包含匹配关键词时，帖子左侧会出现橙色边框，用户名旁出现：

```
[ ☑ 复选框 ] [ ⚠ N keywords ] [ Hide User ] [ 🚫 屏蔽(X) ]
```

1. 点击 **"Hide User"** 按钮
2. 按钮变为 **"Hiding..."** → **"Hidden!"**（绿色）
3. 该用户的所有帖子立即从页面消失
4. 用户被记录到设置页的 **"Hidden Users"** 列表中

### 如何"取消隐藏"？

前往设置页 → **Hidden Users** 部分 → 点击对应用户右侧的 **"Show"** 按钮。

> 注意：取消隐藏后，该用户帖子在刷新页面后重新显示。

## 5. 屏蔽用户（X 原生屏蔽）

### 什么是"X 原生屏蔽"？

点击 **"🚫 屏蔽(X)"** 按钮会自动模拟操作 X 平台的原生屏蔽流程：

1. 自动点击帖子右上角的 **"..."** 菜单
2. 自动选择 **"屏蔽 @username"** 选项
3. 自动点击确认弹窗

这是真正的平台级屏蔽——与你手动操作 X 的效果完全相同，屏蔽记录会同步到 X 账号，对方无法再查看你的内容。

### 操作步骤

1. 在匹配帖子旁点击 **"🚫 屏蔽(X)"** 按钮
2. 按钮变为 **"屏蔽中..."**，此时可以看到 X 的菜单自动弹出并操作
3. 完成后按钮变为 **"✓ 已屏蔽"**（绿色），帖子同时被本地隐藏

> **如果失败**：按钮会恢复到初始状态。常见原因：
> - X 的 DOM 结构发生变化（选择器失效）
> - 页面网络加载过慢，菜单未能及时出现
> - 你已经屏蔽过该用户

## 6. 批量操作

### 6.1 选择用户

每个匹配帖子左侧有一个 **复选框**，勾选后该用户进入批量选择列表。选择 1 个或更多用户后，页面底部出现浮动工具栏：

```
[ 已选择 N 个用户 ] [ 全选 (M) ] [ 隐藏所有选中 ] [ 🚫 屏蔽所有(X) ]
```

### 6.2 全选

点击 **"全选 (M)"** 勾选当前页面所有匹配帖子的用户。

### 6.3 批量 CSS 隐藏

点击 **"隐藏所有选中"** → 并行将所有已选用户进行本地 CSS 隐藏，速度快。

### 6.4 批量 X 原生屏蔽

点击 **"🚫 屏蔽所有(X)"** → 依次对每个用户执行 X 原生屏蔽流程（每个用户间隔 500ms，避免触发 X 的频率限制）。

> **提示**：批量 X 原生屏蔽为顺序执行，用户较多时需要等待。屏蔽期间请勿手动操作页面。

## 7. 管理已隐藏用户

在设置页的 **"Hidden Users"** 部分：

| 操作 | 说明 |
|------|------|
| **搜索** | 在 "Search hidden users..." 输入框中输入用户名快速过滤 |
| **单个显示** | 点击用户右侧的 **"Show"** 取消隐藏 |
| **清空全部** | 点击 **"Clear All"** 并确认，移除所有隐藏记录 |

列表中只记录通过本扩展进行 CSS 隐藏的用户，通过 X 原生屏蔽的用户由 X 平台自行管理。

## 8. 数据备份与恢复

所有数据（关键词 + 已隐藏用户）存储在浏览器本地（`chrome.storage.local`），不上传到任何服务器，**重装浏览器或清除浏览器数据会导致数据丢失**，建议定期备份。

### 导出

设置页 → Keywords Management 部分 → 点击 **"Export Keywords"** → 浏览器自动下载一个 `block-twitter-export-{日期}.json` 文件。

文件内容示例：
```json
{
  "keywords": ["垃圾", "spam", "scam"],
  "blockedUsers": {
    "@example_user": 1714000000000
  },
  "exportedAt": "2026-04-23T00:00:00.000Z",
  "version": "1.0"
}
```

### 导入

设置页 → Keywords Management 部分 → 点击 **"Import Keywords"** → 选择之前导出的 JSON 文件 → 数据**合并**到现有数据中（不会覆盖已有条目）。

### 从 URL 导入

除了本地文件导入外，还可以直接从 URL 导入关键词列表（如 GitHub Gist）。

**支持的格式：**

| 格式 | 示例 | 说明 |
|------|------|------|
| **block-twitter 导出格式** | `{ "keywords": ["spam", "scam"] }` | 与导出文件格式完全相同 |
| **纯 JSON 数组** | `["spam", "scam", "fake"]` | 直接是字符串数组 |
| **纯文本** | `spam\nscam\nfake` | 每行一个关键词 |

**GitHub Gist 操作步骤：**

1. 打开 GitHub Gist（[gist.github.com](https://gist.github.com)），创建新的 Gist
   - 文件内容：每行一个关键词，或 JSON 格式
   - 示例：
     ```
     垃圾
     spam
     scam
     ```

2. 点击 **"Create public gist"** 或 **"Create secret gist"**（都可以，扩展都能访问）

3. 复制 Gist 的**页面 URL**（不是 raw URL，扩展会自动转换）
   - 例如：`https://gist.github.com/yourusername/abc123def456`

4. 回到 block-twitter 设置页 → Keywords Management 部分，找到 **URL 导入**输入框

5. 粘贴 Gist 页面 URL → 点击 **"Import from URL"**

6. 导入成功后，会看到 toast 提示：`Imported N keywords (M new)`
   - N = 此 URL 中找到的有效关键词数
   - M = 新增加到你列表中的个数（去重后）

**URL 导入的优势：**

- 无需手动创建 JSON 文件
- 支持公开分享关键词列表（朋友可以直接复制你的 Gist URL）
- 无需下载文件到本地

## 9. 跨标签页同步

本扩展通过 `chrome.storage.onChanged` 事件实现实时同步：

- 在一个标签页隐藏用户 → 所有打开的 X.com 标签页立即生效，无需刷新
- 在设置页添加/删除关键词 → 所有 X.com 标签页的 MutationObserver 立即以新关键词重新扫描

## 10. 常见问题

### Q：添加关键词后页面没有橙色标记？

1. 确认在 X.com 或 Twitter.com 页面（仅支持这两个域名）
2. 按 F12 打开 DevTools → Console，查找 `[block-twitter]` 开头的日志
3. 尝试刷新页面等待 2 秒（扩展有 2 秒初始化延迟）
4. 在 `chrome://extensions/` 点击扩展的刷新图标，再刷新 X.com

### Q：中文关键词没有效果？

使用子串匹配，确认关键词拼写与帖子内容完全一致（包括空格、标点）。中文不受大小写影响。

### Q：含 Emoji 的关键词没有效果？

确保关键词中的 Emoji 与帖子中的完全一致（同一个 Emoji 的不同变体码位可能不同）。建议直接复制帖子中的 Emoji 到关键词输入框。

### Q："🚫 屏蔽(X)" 按钮一直显示"屏蔽中..."然后恢复原样？

X 的屏蔽菜单未能找到。可能原因：
- X 更新了 UI 结构，菜单选择器失效（可查看 DevTools Console 中的 TrueBlocker 警告日志）
- 网络较慢导致菜单加载超时（3 秒超时）
- 已经屏蔽过该用户

### Q：刷新页面后被隐藏用户的帖子又出现了？

已隐藏用户数据保存在本地存储中，页面刷新后 MutationObserver 会重新拦截帖子并隐藏。若帖子在扩展初始化（约 2 秒内）前已渲染，可能短暂闪现。这是已知限制，与 X.com 动态加载机制有关。

### Q：数据保存在哪里？会上传到云端吗？

所有数据存储在浏览器的 `chrome.storage.local` 中，完全本地化，不会上传到任何服务器。使用 DevTools → Application → Storage → chrome-extension://[ID]/ 可以直接查看原始数据。
