# 外部 AI API：分类输出解析增强 + 输出格式约束

日期：2026-04-27  
范围：`src/background/background.js`（`handleAIClassify`）

## 背景 / 问题

外部 API 模式下，扩展会把一组帖子发给模型，并要求模型逐条输出 `spam/ok`。  
但当前解析逻辑非常严格：

- 只匹配以 `"<index>:"` 开头的行（例如 `1: spam`）
- 如果模型输出 `1：spam`（全角冒号）、`1. spam`、`1 - spam`、`1) spam` 等常见变体，就会解析失败
- 解析失败会默认当作 `ok`，表现为“明显垃圾也没识别出来”

因此在提示词已经较强的情况下，仍可能出现大量漏报，但根因是 **解析器不兼容**。

## 目标 / 成功标准

1. 解析兼容多种分隔格式：`: / ： / . / - / )` 等
2. 兼容 `spam/ok` 大小写与前后空格
3. 提示词（userMsg）强制约束输出格式，优先引导模型输出半角冒号 `:`，减少解析歧义
4. 默认不输出敏感 debug 信息（可选加开关，但非必须）

## 非目标

- 不改变模型后端选择逻辑（是否换模型作为后续可选项）。
- 不改变 UI 与数据结构。

## 方案（采纳）

### 1) 增强解析器

把：

```js
const line = text.split('\\n').find(l => l.trimStart().startsWith(`${i + 1}:`));
return line?.toLowerCase().includes('spam') ? 'spam' : 'ok';
```

改为正则匹配（示例）：

- 接受 `1:` `1：` `1.` `1)` `1 -` `1–` 等
- 捕获 `spam|ok` 作为结果

```js
const re = new RegExp(`^\\\\s*${i + 1}\\\\s*[:：.)\\\\-–—]\\\\s*(spam|ok)\\\\b`, 'i');
const line = text.split(/\\r?\\n/).find(l => re.test(l));
const m = line?.match(re);
return (m?.[1] || 'ok').toLowerCase() === 'spam' ? 'spam' : 'ok';
```

### 2) 强制输出格式（userMsg）

将 `userMsg` 文本改得更强约束，并给出 2 行示例，明确要求“只输出 N 行、每行必须用半角冒号 `:`”。

例如：

```text
对以下帖子分类。只输出 N 行，每行严格使用半角冒号 ':'，格式必须为: "序号: spam" 或 "序号: ok"。
不要输出任何解释、标题、空行或多余字符。
示例:
1: spam
2: ok
```

## 手动测试

1. 用同一批回复区垃圾内容扫描：
   - 若模型输出格式有偏差，解析器也能正确识别
   - 命中率应明显提升（此前“全 ok”场景应改善）
